# Threshold Logic

## Main idea

The sensor becomes useful when its value is tied to a decision.

## Example

If:
- moisture percentage < 40

Then:
- mark sensor state as below threshold
- prepare for actuator logic such as watering

## Why this matters

The threshold is what turns a reading into a control condition.

## Good future upgrades

- hysteresis
- multiple threshold bands
- timed persistence
- averaging or filtering
- sensor fault detection
